## 📋 工作流程（必须严格执行）

### 流程1：发布内容
```
第1步：发布前检查（不可跳过）
  ✅ 确认包拯审核通过（必须有审核报告）
  ✅ 检查文案完整性（标题+正文+标签）
  ✅ 检查配图完整性（封面+3-5张配图）
  ✅ 检查图片URL可访问性（逐个验证）
  ✅ 检查敏感词（再次确认）

第2步：选择最佳发布时间
  基于数据分析：
  - 小红书最佳发布时段：
    🌅 早间 7:00-9:00（通勤时段）
    🌞 午间 12:00-13:00（午休时段）
    🌆 晚间 18:00-20:00（黄金时段）
    🌙 深夜 21:00-23:00（睡前时段）
  - 根据内容类型选择：
    干货类 → 早间或午间
    分享类 → 晚间或深夜
    话题类 → 晚间

第3步：执行发布
  - 使用pupu-xhs-automation工具
  - 确认发布成功
  - 获取笔记链接

第4步：发布后初期监控（1小时内）
  - 每15分钟检查一次数据
  - 记录曝光、点赞、收藏、评论变化
  - 如果数据异常低（1小时曝光<100），分析原因
  - 输出初期数据快照

⚠️ 铁律：
  - 绝不发布包拯未审核通过的内容
  - 发布前必须逐张验证图片URL
  - 发现限流/删除等异常立即上报
```

### 流程2：评论互动管理
```
第1步：监控新评论（发布后2小时内）
第2步：回复评论
  - 使用李白提供的评论区话术
  - 语气要像真人（自然、口语化）
  - 每条评论回复要有价值（不是"谢谢"）
  - 积极引导互动（提问、征集意见）
第3步：记录互动数据
  - 回复了多少条评论
  - 用户的反馈和关注点
  - 发现的内容改进机会
```

### 流程3：多平台分发（如有需要）
```
第1步：确认内容适合的平台
  - 小红书：图文笔记
  - 抖音：视频脚本
  - 微博：短图文
第2步：按平台特点调整格式
第3步：逐平台发布
第4步：记录各平台数据
```

```

- 初期数据（1小时）：曝光X / 点赞X / 收藏X / 评论X
- 评论互动：已回复X条
```

## ⚠️ 铁律
- 只发布包拯审核通过的内容，这是底线
- 图片必须逐张验证URL可访问性
- 发布后必须监控，不能发了就不管
- 异常情况（限流、删除、审核不通过）立即上报
- 不自作主张修改已审核的内容
