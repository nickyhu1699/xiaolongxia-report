## 🎨 公众号文章排版优化 - CSS设计原则

### 排版问题诊断与解决

#### 问题1：间距过大
- **症状**：段落之间、标题与正文之间间距过大
- **根本原因**：
  - 使用全局 `line-height: 1.4` 影响所有元素
  - p标签有默认的 `margin-top` 和 `margin-bottom`
  - 外层容器的 `padding: 15px` 过大

#### 问题2：引用块布局不协调
- **症状**：引用块（有背景色）的文字和边缘挤压
- **根本原因**：
  - 引用块同时使用 `padding` 和 `margin`，导致双重间距
  - `padding` 不够大（12px），文字和边缘没有足够空间

### CSS设计原则（重要学习）

#### 1. 为不同元素类型分别定义line-height
```css
/* ❌ 错误：全局定义 */
line-height: 1.4;

/* ✅ 正确：分别定义 */
.title { line-height: 1.2; }      /* 标题：紧凑 */
.subtitle { line-height: 1.3; }   /* 子标题：适中 */
.paragraph { line-height: 1.6; }  /* 段落：保证可读性 */
.list-item { line-height: 1.4; }  /* 列表项：紧凑 */
.quote { line-height: 1.5; }      /* 引用块：适中 */
```

**原因**：全局 `line-height` 会影响所有元素，包括标题、段落、列表项，导致排版不协调。

#### 2. 引用块只需要padding，不需要margin
```css
/* ❌ 错误：同时使用padding和margin */
.quote {
    padding: 10px 12px;
    margin-bottom: 15px;
    margin-top: 2px;
}

/* ✅ 正确：只用padding提供视觉分隔 */
.quote {
    padding: 10px 15px;
    margin: 0;  /* 移除所有margin */
}
```

**原因**：引用块已经有背景色，`padding` 可以提供内部间距，不需要额外的 `margin`。

#### 3. 移除p标签的默认margin
```css
/* 重置默认margin */
p, h2, h3 { margin: 0; }

/* 然后精确控制每个元素的间距 */
.paragraph { margin-bottom: 6px; }
.list-item { margin-bottom: 2px; }
```

### 最终优化版本（按照公众号文章模板）

#### 样式规范
```css
/* 标题 */

    font-size: 22px;
    font-weight: bold;
    color: #333333;
    line-height: 1.5;
    text-align: center;
    margin-bottom: 30px;

}

/* 正文段落 */

    line-height: 1.8;
    margin: 16px 0;
    text-align: justify;
}

/* 引用块 */

}

/* 强调色 */
strong { color: #e6544a; }  /* 红色 */

}
```
