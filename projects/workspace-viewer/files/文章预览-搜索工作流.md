#### 1. 搜索策略决策树（400字）
```
用户要求搜索 → 
  ├─ 快速搜索？ → web_search（1-2秒）
  ├─ 需要完整内容？ → web_fetch（5-10秒）
  ├─ 特定平台？ → agent-reach
  └─ 复杂网页？ → browser（10-30秒）
```

**说明**：

#### 2. 4种搜索方式详解（1200字）

**方式1：web_search - 闪电搜索（200字）**
- 速度：1-2秒
- 适用场景：快速了解全局
- 实际案例：搜索"江门美食攻略"
- 优点：最快
- 缺点：只有摘要

**方式2：web_fetch - 深度阅读（200字）**
- 速度：5-10秒
- 适用场景：获取完整内容
- 实际案例：深度阅读美食文章
- 优点：完整内容
- 缺点：较慢

**方式3：agent-reach - 平台专属（400字）**
- 速度：5-15秒
- 支持平台：16个
- 重点介绍：
  - 小红书搜索
  - B站视频搜索
  - Twitter搜索
  - 微信公众号
- 实际案例：搜索小红书"咖啡店活动"

**方式4：browser - 浏览器自动化（200字）**
- 速度：10-30秒
- 适用场景：复杂网页、需要JavaScript渲染
- 优点：完整渲染
- 缺点：最慢

#### 3. 实战案例：搜索江门美食攻略（400字）
- 第一步：web_search（1-2秒）→ 获取搜索结果列表
- 第二步：web_fetch（5-10秒）→ 深度阅读重点文章
- 第三步：整理和分析 → 生成完整攻略
- 成果展示：30条美食信息、5条美食街
